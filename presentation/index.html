1 — Simple LangGraph Agent

create_react_agent: LLM ↔ Tools loop. Minimal setup.

from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver
from langchain_core.tools import tool

_store = {}

@tool
def write_data(key: str, value: str) -> str:
    """Write a key-value pair to the data store."""
    _store[key] = value
    return f"Stored: {key!r} = {value!r}"

@tool
def read_data(key: str) -> str:
    """Read a value from the data store."""
    return _store.get(key, f"Not found: {key!r}")

llm = ChatOllama(model="qwen3:8b", base_url="http://10.0.0.224:11434")
agent = create_react_agent(llm, [write_data, read_data],
                           checkpointer=MemorySaver())
LangGraph manages the ReAct loop: LLM → tool call → result → LLM → ...

4 — Human-in-the-Loop

from langgraph.types import interrupt

@tool
def request_options(
    question: str,
    options: list[str],
    config: RunnableConfig
) -> str:
    # Pauses the graph here.
    # State is saved to checkpointer.
    # GUI receives "interrupt" event.
    choice = interrupt({
        "question": question,
        "options": options
    })
    return f"Human chose: {choice}"
# Resume with human's choice:
from langgraph.types import Command

agent.invoke(
    Command(resume="option A"),
    config={"configurable": {
        "thread_id": "abc123"
    }}
)

6 — Aegra: LangGraph Platform API

# Implements the LangGraph Platform REST spec
# (same API that official LangSmith runtime exposes)

POST  /threads                    → { thread_id, status }
GET   /threads/{id}/state         → full state snapshot
POST  /threads/{id}/runs/stream   → SSE with typed events

# SSE event names are standardised:
event: metadata     data: { "run_id": "...", "assistant_id": "..." }
event: updates      data: { "node": "agent", "state": {...} }
event: messages/partial  data: { "content": "token..." }
event: tool_start   data: { "tool": "write_code", "input": {...} }
event: interrupt    data: { "run_id": "...", "value": {...} }
event: values       data: { ...full final state... }
event: end          data: {}
Front-end uses: event === "messages/partial" — typed event name in SSE header, not a field inside data. Compatible with the official @langchain/langgraph-sdk.

8 — Platform Strategy

🔍 Glean
Wrap as a System Tool inside the Deep Agent. search_knowledge() calls Glean API, LLM synthesizes answer. Keeps context window clean via VFS.
🌊 LangFlow
Create a Custom Component wrapping the LangGraph SDK client. Low-code users drag one "Agent" node; pro-code edit graph internals.
☸ Kubernetes
Aegra = FastAPI + Postgres. Runs in any K8s cluster. No LangSmith license needed. Standard Dockerfile + Helm chart.
☁ Azure Foundry
Skip "Prompt Agents" (too limited). Use Azure Hosted Agents (Docker) or K8s + Aegra path for full polymorphic control.

Summary

Deep Agent = LangGraph + opinionated body
VFS (offload artifacts), write_todos (planning), skills (specialized LLMs), HITL (interrupts)
FastAPI → Aegra migration path
Start raw, grow into the Platform API as teams scale. Same LangGraph code, different host.
React GUI pattern
Token streaming + tool spinners + VFS panel + HITL modal. FastAPI needs raw fetch; Aegra can use the SDK.
No vendor lock-in
Ollama (local), Aegra (Apache 2.0), LangGraph (MIT), FastAPI (MIT). K8s-native.
